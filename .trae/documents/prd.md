# 政府类科研项目申报智能体 — 前端 PRD

## 一、产品定位

**产品名称**：智策 · 政府科研项目申报助手（GovDecl Agent Web）

**目标用户**：
- 高校 / 科研院所的科研人员
- 医院、企业研发部门的项目申报专员
- 各级科技项目管理单位、机关单位的科技处

**核心价值**：
让科研工作者从"读通知、找模板、写申报书、查政策、补材料"的重复劳动中解放出来，专注科研本身。智能体可以理解各级别（国家自然基金委、科技部、工信部、省市级）的申报指南，结合用户上传的项目基础材料（论文、专利、可行性报告等）与内部知识库，**自动生成符合规范的科研项目申报书**，并支持多轮修订与版本管理。

## 二、典型使用场景

| 场景 | 描述 | 智能体能力 |
| --- | --- | --- |
| **政策解读** | 用户粘贴一段 2026 年度国家自然科学基金申报指南 | 智能体解析要点、给出选题建议、注意事项 |
| **选题立项** | "我打算做基于多模态大模型的 xxx 研究，可行吗？" | 分析研究背景、国内外现状、立项依据 |
| **申报书撰写** | "请帮我撰写《项目研究内容与关键科学问题》一节" | 调用知识库，结合已有材料，按模板结构生成 3000-5000 字草稿 |
| **预算编制** | "本项目总经费 80 万，请帮我编制预算明细" | 按经费科目分类输出符合规定的预算表 |
| **材料审校** | 用户上传申报书 PDF | 逐条对照指南检查格式、字数、签字盖章项 |
| **答辩 PPT** | "基于现有申报书，生成答辩 PPT 大纲" | 自动提炼 15 页答辩结构与讲稿 |

## 三、用户旅程

```
登录 → 工作台 → 选择/创建任务（按基金类型）→ 上传基础材料（可选）
       ↓
新建对话 → 输入需求 → 流式生成（含思考过程）→ 引用知识库出处
       ↓
查看草稿（在右侧面板）→ 标记采纳 / 修改 / 重写
       ↓
导出 Word / PDF → 评审预演 → 提交
```

## 四、关键功能需求

### 4.1 对话主区（核心）
- **流式对话**：基于 LangGraph 的 SSE 流式输出，仿主流 AI Agent（ChatGPT / Claude / Kimi）体验
- **思考过程展示**：折叠式"思维链"区域，让用户看到智能体的工具调用与推理步骤
- **文件引用**：智能体回答中显示引用的知识库条目（脚注 + 内联引用）
- **多模型切换**：GPT-4、Claude、DeepSeek、Qwen、智谱 GLM、文心一言等
- **多模式切换**：普通模式 / 计划模式（自动拆解任务为待办列表）
- **停止生成 / 重新生成 / 复制 / 点赞点踩**

### 4.2 左侧栏
- **新对话**按钮
- **对话历史**列表（按时间分组：今天 / 昨天 / 上周 / 更早）
- **收藏** / 固定对话
- **按基金类型筛选**：国家自然基金 / 重点研发 / 省自然基金 / 横向
- **导入 / 导出** Markdown

### 4.3 顶部栏
- 项目品牌 logo + 名称
- 顶部主导航：**对话 / 知识库 / 设置**（单页切换）
- 全局搜索（Ctrl+K）
- 主题切换按钮（亮 / 暗）
- 当前模型指示（点击进入"设置 → 大模型"修改）

> ❌ 移除原"右侧栏"：知识库、申报草稿、文件、计划 4 个 Tab 全部迁移到独立页面或行内抽屉。

### 4.4 知识库（独立页面）
- 顶部：检索框、分类筛选（全部 / 政策文件 / 申报模板 / 历史中榜 / 内部资料 / 我的上传）、视图切换（列表 / 网格）、批量上传
- 左侧：分类导航树
- 主区：知识条目列表（含类型徽章、来源、更新时间、标签、引用次数）
- 右侧：选中条目的内嵌预览（PDF 缩略图 / Markdown 渲染 / 元信息）
- 操作：新建、上传、批量整理、建立索引

### 4.5 设置（独立页面）
左侧设置分组导航 + 右侧设置表单：
1. **大模型**：当前模型选择（DeepSeek / Qwen / GLM / Claude / GPT / 文心一言 / 自定义）、温度、最大输出 token、top_p、推理深度、是否启用 thinking
2. **智能体能力**（开关矩阵）：
   - 计划模式（Plan Mode / TodoList）
   - 深度思考（Thinking）
   - 子智能体委派（Subagent）
   - 长期记忆（Memory）
   - 联网搜索（Web Search）
   - 知识库检索（Knowledge RAG）
   - 文档转换（PDF/Word/Excel/PPT）
   - 工具护栏（Guardrail）
3. **技能 / Skills**：已启用技能列表 + 从 `.skill` 安装
4. **MCP 服务**：MCP 服务器配置（stdio / sse / http + OAuth）
5. **外观**：主题（亮 / 暗 / 跟随系统）、主色微调、字号、字体
6. **通用**：流式输出、Markdown 渲染、快捷键、消息历史清理

### 4.6 底部输入区
- 多行自适应文本框（Enter 发送，Shift+Enter 换行）
- 附件上传按钮（支持 PDF / Word / Excel / PPT / 图片）
- 工具开关：网络搜索、知识库检索、计划模式
- 字数计数、模型 Token 实时统计

### 4.7 行内抽屉（仅对话页）
- 草稿面板：折叠式右侧抽屉，列出当前任务的所有章节
- 计划面板：折叠式右侧抽屉，展示 todo 列表
- 默认收起，可通过顶栏图标展开

## 五、信息架构

```
/
├── 对话（/chat，默认）
│   ├── 左侧栏（对话历史）
│   ├── 主对话区
│   └── 行内面板（草稿 / 计划，可折叠抽屉）
├── 知识库（/knowledge）   ← 独立全屏页面
│   ├── 检索 / 筛选
│   ├── 列表 / 详情双栏
│   └── 上传 / 整理
└── 设置（/settings）     ← 独立全屏页面
    ├── 大模型
    ├── 智能体能力（技能 / MCP / 记忆 / 计划模式 / 子智能体 / 护栏 / 联网）
    ├── 工具与扩展
    └── 通用
```

❌ **本期不交付**：用户登录、鉴权、用户菜单（无账号系统）。

## 六、设计语言

### 6.1 视觉关键词
**专业 · 庄重 · 智慧 · 现代**

- 借鉴主流 AI Agent 产品（ChatGPT / Claude / Kimi / 文心一言）的简洁三栏布局
- 借鉴政务系统的庄重感，使用 **深靛蓝 + 朱砂红 + 宣纸米色** 的中国政务配色
- 借鉴现代设计趋势：圆角、玻璃拟态、细腻的微动画

### 6.2 色彩系统（已替换为新版主题）

摒弃传统政务红蓝配色，选用**森林苔藓 + 暖陶土** 的低饱和度科研配色，体现"理性、温度、可持续"。

#### 6.2.1 浅色模式（默认）
- **主色**：`#1F4A36`（深森林绿）—— 庄重且不沉闷
- **辅色**：`#C26A4D`（暖陶土）—— 强调、按钮、徽章
- **点缀**：`#D4A574`（古铜金）—— 重要徽章、印章风角标
- **背景**：`#FAF6EE`（暖象牙）
- **卡片**：`#FFFFFF`
- **墨色**：`#1A1F1B`（带绿意的深墨）
- **雾灰**：`#7A8480`
- **描边**：`#E8E2D2`

#### 6.2.2 深色模式
- **主色**：`#9BC4A0`（柔鼠尾草）
- **辅色**：`#E8956D`（暖珊瑚）
- **点缀**：`#E8B568`（琥珀金）
- **背景**：`#0E1410`（深炭墨）
- **卡片**：`#161D17`
- **墨色**：`#E5E8E2`（米白）
- **雾灰**：`#7E8A82`
- **描边**：`#252D26`

> 两种模式均保证正文对比度 ≥ 7:1（AAA 级）。

### 6.3 字体
- 中文：思源宋体（标题，体现庄重）+ 思源黑体（正文）
- 英文 / 数字：Inter 或 SF Pro
- 代码：JetBrains Mono

### 6.4 视觉细节
- 左侧栏：可折叠、显示项目 Logo 印章风格的角标
- 对话气泡：用户靠右、AI 靠左，AI 气泡带"智能体"印章角标
- 加载状态：流式光标 + 思维链骨架屏
- 顶部装饰：极细的中国传统回形纹（`#`字符的延伸）作为分隔线
